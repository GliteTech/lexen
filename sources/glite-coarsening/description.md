# Glite Coarsening Subset

This directory contains the public Glite coarsening subset needed for the lexEN release label layer
and the three-reviewer agreement report. The map is limited to WordNet sense keys referenced by the
Maru2022/lexEN candidate inventories, source labels, and lexicographer review labels.

## Coarsening Policy

The release and report map each selected WordNet sense key to a Glite concept ID. Multi-label
answers are coarsened to a sorted set of concept IDs. If a sense key is not present in the public
map, the coarsener first checks the explicit alias file. If no alias exists, it keeps an explicit
`unmapped:<sense_key>` marker instead of guessing a concept. This preserves auditability for WordNet
keys that do not have a public Glite coarsening row.

The alias file is deliberately small. It only covers keys needed to reproduce the agreement report
where the equivalence is direct and auditable: British `organisation` keys whose American-spelling
`organization` keys are in the Glite map, plus two legacy WordNet keys (`court%1:06:05::` and
`show%1:10:00::`) that map to already-present Glite concepts for the same meaning.

`cannot_answer` decisions stay separate from sense-label decisions in the agreement report. This
makes the coarse agreement tables auditable and prevents unanswerable items from being folded into
any lexical concept.

## Provenance

The public subset is generated by
[`scripts/build_public_glite_subset.py`](../../scripts/build_public_glite_subset.py). The script
derives the required WordNet sense-key set from the committed Maru2022/lexEN source artifacts and
writes only four public fields per mapped row: `sense_key`, `lemma`, `wordnet_pos`, and
`concept_id`.
